# 项目化学习模式

## 选择原则

- 编程主题优先做能运行、能调试、能扩展的东西
- 非编程主题优先做能表达、能分析、能迁移的产出
- 产出规模以“本轮能完成并得到反馈”为准，不求大而全

## 编程主题模板

适用：编程语言、框架、数据分析、算法、系统设计、AI 工程。

可选产出：

- 最小功能 demo
- 一段 bug 修复或重构任务
- 一个接口、页面或工作流的小任务书
- 一个微项目里程碑

任务书至少包含：

- 目标
- 输入输出
- 限制条件
- 验收标准
- 一个变式挑战

## 非编程主题模板

适用：数学、经济、历史、语言、心理学、写作、演讲、管理等。

可选产出：

- 研究短报告
- 讲解稿或演示提纲
- 读书笔记
- 案例分析
- 知识地图
- 对比表
- 口语表达脚本
- 复盘文档

任务设计至少包含：

- 本轮主题
- 产出格式
- 目标受众
- 使用场景
- 评价标准

## 递进方式

推荐按这个梯度推进：

1. 微产出：5 分钟到 20 分钟内可完成
2. 小项目：1 次学习会话内可完成
3. 阶段项目：多次会话串联，形成作品或系统成果

## 检查点设计

每个项目都至少要有一个检查点：

- 能否复述目标
- 能否解释关键选择
- 能否指出一个常见错误
- 能否做一个小迁移
