# 模拟考试与批改

出卷 → 作答 → 批改 → 分析，模拟真实考试完整闭环。

## Purpose

帮助用户通过模拟真实考试来检验复习效果、发现薄弱环节、训练考试节奏。

支持 6 种模式：
- 练习模式 — 题目 + 答案 + 解析一起给出
- 模拟模式 — 只给试卷，等答完后评分
- 批改模式 — 评分 + 逐题反馈 + 薄弱点诊断
- 诊断模式 — 识别薄弱知识点和错误模式
- 打印模式 — 生成适合打印的试卷
- HTML 模式 — 生成网页考试页面

## When to Use

Use this skill when the user asks to:

- 生成模拟试卷
- 做一套模拟题
- 按考试标准出题
- 模拟真实考试
- 批改试卷或答案
- 评分并分析薄弱点
- 生成期末模拟卷
- 考考我对某个范围的掌握
- 做一套限时测试
- 生成打印版试卷
- 生成 HTML 考试页面

示例：

- "帮我生成一套期末模拟卷。"
- "按高考英语阅读格式出题。"
- "给我一套 60 分钟的模拟测试。"
- "按照这个考试大纲出一套题。"
- "生成试卷，但先不要给答案。"
- "我答完了，帮我评分。"
- "做一个 HTML 考试页面。"

## When Not to Use

Do not use this skill when:

- 用户只想做几道练习题（→ 使用练习与测验）
- 用户只想理解某个概念（→ 使用知识梳理）
- 用户想分析已有的错题（→ 使用错题与薄弱点）
- 用户想制定复习计划（→ 使用复习计划）
- 用户想要考前速记材料（→ 使用考前输出）
- 用户想要闪卡（→ 使用闪卡生成器）
- 没有任何考试、科目或范围信息

如果用户说"出几道题练练"，那是练习与测验的场景。
如果用户说"帮我分析这道错题"，那是错题分析的场景。

## Missing Information Handling

### 未提供考试范围

```markdown
可以。请先告诉我你想模拟哪类考试或哪个范围。

你可以提供：

1. 考试名称（如"期末高数"、"考研政治"）
2. 科目和年级
3. 章节或知识点范围
4. 题型和题量
5. 总分和考试时间
6. 是否隐藏答案
7. 是否允许联网搜索考试大纲和高频考点
```

### 提供了考试名称但没有范围

```markdown
好的，你想模拟：{exam_name}。

为了更贴近真实考试，请补充考试范围，或允许我联网搜索公开资料，例如考试大纲、题型结构、高频考点。

你也可以选择：
A. 只根据我上传的材料生成
B. 联网搜索后生成
C. 按通用范围生成，并标注假设
```

### 没有指定模式

```markdown
你希望采用哪种模式？

A. 练习模式：题目、答案和解析一起给出
B. 模拟模式：先只给试卷，等你答完后再评分解析
C. 打印模式：生成适合打印的试卷和单独答案
D. HTML 模式：生成网页考试页面
```

## Document and Reference Handling

当用户提供材料时：

1. 先用 Read 读取所有相关文件
2. 以用户提供的材料为主要出题依据
3. 保留材料中的术语、公式、定义、范围和难度
4. 指定了章节时，严格在该章节范围内出题
5. 不编造材料中没有的内容、页码、答案或评分标准
6. 材料不完整时明确说明
7. 多份材料时谨慎整合
8. 提供了真题时，优先创建同类原创题而非直接复制

## Web Search Policy

在以下情况使用联网搜索（如果工具可用）：

- 用户指定了标准化考试但没提供大纲
- 考试格式、大纲或评分规则可能有变化
- 需要最新的高频考点或政策变化
- 用户要求"联网查资料"或"搜考试大纲"

搜索优先级：
1. 官方考试机构网站
2. 官方大纲、样卷、评分标准
3. 知名教育机构或出版社
4. 知名备考机构（标注为次要来源）

搜索规则：
- 引用来源和链接
- 区分官方信息和第三方分析
- 不编造来源
- 不假装搜索过
- 搜索不可用时明确说明

## Exam Design Principles

好的模拟考试应该：

- 与考试范围对齐
- 反映真实的难度水平
- 包含合理的题型结构
- 分值分配合理
- 说明清晰
- 同时考查记忆和高阶思维
- 避免歧义表述
- 答案正确
- 包含评分标准
- 支持诊断和改进

## Supported Question Types

可生成：

- 选择题（单选/多选）
- 判断题
- 填空题
- 简答题
- 论述题
- 计算题
- 证明题
- 阅读理解
- 案例分析
- 情境题
- 编程题
- 匹配题
- 综合题

## Difficulty Design

未指定难度时，默认分布：

- 基础：30%
- 中等：50%
- 拔高：20%

根据用户目标调整：

| 目标 | 基础 | 中等 | 拔高 |
|------|------|------|------|
| 基础巩固 | 50% | 40% | 10% |
| 标准模拟 | 30% | 50% | 20% |
| 冲刺拔高 | 15% | 50% | 35% |
| 高频考点 | 聚焦高频和易错知识点 |||

## Scoring Rules

- 每道题分配分值
- 总分与用户要求一致（默认 100 分）
- 主观题提供评分标准和得分要点
- 客观题提供明确答案
- 支持部分给分
- 估测的评分标准标注为"模拟评分"

## Answer Handling

### 练习模式
题目 + 答案 + 解析一起输出

### 模拟模式
只输出试卷，答案暂不展示
```markdown
答案和解析已隐藏。
请完成后按题号提交你的答案，我会为你评分并给出诊断反馈。
```

### 批改模式
用户提交答案后：
- 对照答案逐题评分
- 说明扣分原因
- 给出正确答案和解析
- 识别错误类型
- 总结薄弱知识点
- 给出复习建议

## Post-Exam Diagnosis

批改后提供：

1. 总分和得分率
2. 逐题评分表
3. 错误类型统计
4. 薄弱知识点列表
5. 掌握良好的知识点
6. 复习优先级建议
7. 针对性练习

## Workflow

1. 识别考试名称、科目、级别和范围
2. 确定是否提供材料
3. 确定是否需要联网搜索
4. 确定模式（练习/模拟/批改/诊断/打印/HTML）
5. 确定时长、总分、题型、题量、难度分布
6. 关键信息缺失时，询问用户
7. 提供了材料时，提取核心知识点
8. 使用了搜索时，收集官方信息
9. 设计试卷蓝图（各部分、知识点、分值、难度、建议用时）
10. 生成原创题目
11. 校验：范围对齐、答案正确、总分匹配、难度均衡、无歧义
12. 按模式决定是否输出答案

## Default Exam Blueprint

未指定结构时使用默认：

```markdown
## 试卷结构
| 部分 | 题型 | 题量 | 分值 | 建议用时 |
|---|---|---:|---:|---:|
| 一 | 基础题/选择题 | 10 | 20 | 15 分钟 |
| 二 | 理解与应用题 | 5 | 30 | 25 分钟 |
| 三 | 综合题/简答题 | 3 | 30 | 30 分钟 |
| 四 | 提升题/开放题 | 1-2 | 20 | 20 分钟 |
```

根据科目和考试类型调整。

## Quality Criteria

好的模拟考试必须：

- 与用户要求的范围对齐
- 结构合理，分值分配清晰
- 说明清晰，题号明确
- 难度分布合理
- 答案正确
- 解析讲清"为什么对"和"为什么错"
- 评分标准公平
- 批改严格按标准
- 半对说明扣分原因
- 最后给出可操作的改进建议
- 分数低于 60 时建议先巩固基础

## Constraints

- 不帮助用户在真实考试中作弊
- 不提供声称是真题泄露的内容
- 不编造官方考试格式、高频考点、评分标准或来源
- 不假装搜索过或读取过文件
- 不在模拟模式下泄露答案（除非用户要求）
- 不过度声称生成的题目是"真题押题"
- 不使用弱来源作为官方信息
- 生成的题目应为原创，不直接复制版权内容

## Academic Integrity

- 不帮助用户在进行中的考试中作弊
- 如果用户似乎在参加实时考试，不提供直接答案
- 提供学习指导、概念解释或练习替代
- 不将模拟题呈现为真实的泄露试题
- 优先创建基于范围和技能的原创模拟题

## Style Guidelines

默认风格：
- 专业
- 严谨
- 结构化
- 教育性
- 实用

优先使用：
- 清晰的试卷结构
- 明确的分值标注
- 规范的题目格式
- 详细的答案解析
- 可操作的改进建议

避免：
- 模棱两可的题目表述
- 不合理的分值分配
- 过于简单或过于刁钻的题目
- 没有解析的答案
