# 解题工作流

面对一道题目时的标准解题流程。适用于所有学科。

## 通用解题流程（UPER模型）

### U — Understand（理解题目）

1. **通读题目**：完整读一遍，不跳过任何字
2. **标记关键信息**：
   - 已知条件（画下划线）
   - 求解目标（画圈）
   - 关键约束（画框）
3. **用自己的话重述**："这道题是让我……已知……求……"
4. **识别题型**：这属于哪类问题？以前见过类似的吗？
5. **确认理解**：有没有不确定的术语或条件？

⚠️ 常见错误：没读完题目就开始算——漏掉关键条件

### P — Plan（规划方案）

1. **关联知识**：这道题考的是什么知识点？
2. **选择方法**：可能有哪些解法？选哪个最合适？
3. **拆解步骤**：把问题拆成2-5个子步骤
4. **预判难点**：哪一步可能会出问题？
5. **验证可行性**：这个计划能到达目标吗？

⚠️ 常见错误：想到一种方法就冲——可能不是最优解

### E — Execute（执行计算）

1. **逐步执行**：按照计划一步一步来
2. **每步检验**：每完成一步，检查是否合理
   - 量纲对不对？（物理/化学）
   - 数量级对不对？（数学）
   - 逻辑通不通？（所有学科）
3. **记录过程**：写下关键步骤，不要只写答案
4. **处理卡点**：卡住时不死磕，回到Plan换思路

⚠️ 常见错误：算错一步导致全盘皆错——每步都要检验

### R — Review（回顾反思）

1. **检验答案**：答案合理吗？能代入验证吗？
2. **回顾过程**：
   - 哪里花的时间最多？为什么？
   - 有没有更简洁的解法？
   - 错误出在哪个环节？
3. **归纳总结**：
   - 这道题的核心考点是什么？
   - 同类题的通用解法是什么？
   - 可以用什么技巧避免同类错误？
4. **举一反三**：如果条件变了，解法怎么调整？

⚠️ 常见错误：做对就完了——不反思就不知道为什么对，下次可能就错

## 分学科解题工作流

### 数学解题

1. U：识别题型（求导/积分/证明/应用），标记所有条件
2. P：选择方法（直接法/间接法/反证法/构造法），列出需要的公式
3. E：按步骤计算，每步保留中间结果，注意特殊值和边界
4. R：代回验证，检查是否遗漏条件，总结题型套路

### 代码调试

1. U：理解报错信息，复现bug，确定预期行为vs实际行为
2. P：定位策略（二分法/打印法/断点法），推测可能原因（排序1-3）
3. E：从最可能的原因开始排查，每次只改一个变量
4. R：确认修复，添加测试防止回归，记录根因

**调试深度策略**：

| 错误类型 | 识别特征 | 调试方法 |
|---------|---------|---------|
| 语法错误 | 编译器/解释器报错 | 读错误信息，定位行号 |
| 运行时错误 | 程序崩溃、异常 | 堆栈追踪，断点调试 |
| 逻辑错误 | 输出不对但没崩溃 | 打印中间值，与预期对比 |
| 性能错误 | 运行太慢 | 性能分析(profiler)，找瓶颈 |
| 并发错误 | 间歇性出错 | 竞态条件分析，加锁检查 |

**AI辅助调试原则**：不直接给修复代码，而是引导学生：
1. "错误信息说的是什么意思？"
2. "你觉得哪一行可能有问题？"
3. "如果在这行加个print，你觉得会输出什么？"
4. "试试把这段代码简化到最小可复现案例"

### 英语阅读理解

1. U：先读题目再看文章（带着问题读更高效）
2. P：定位关键词，确定题目类型（主旨/细节/推断/词义）
3. E：回到原文定位，对比选项和原文表述
4. R：检查是否被干扰项误导，总结出题规律

### 物理解题

1. U：画出示意图，标注已知量和未知量，明确物理过程
2. P：选择物理模型和定律，列出方程
3. E：解方程，注意单位换算，检验量纲
4. R：检查数量级是否合理，考虑极端情况

### 化学解题

1. U：识别反应类型，写出化学方程式
2. P：选择解题路径（守恒法/差量法/关系式法）
3. E：计算时注意化学计量数，检查单位
4. R：验证质量/电荷/原子守恒

### 历史分析题

1. U：审题——是原因分析/影响评价/比较论述？
2. P：构建答题框架（政治/经济/文化多角度）
3. E：论点+史实支撑，逻辑递进
4. R：检查是否有史实错误，论证是否完整

## AI辅助策略

AI在解题工作流中的角色不是"代做"，而是"引导"：

| 步骤 | AI应该做 | AI不应该做 |
|------|---------|-----------|
| Understand | 帮学生检查是否遗漏条件 | 直接告诉学生"这题考的是XX" |
| Plan | 用苏格拉底法引导学生思考方案 | 直接给出完整解题步骤 |
| Execute | 在学生卡住时给局部提示 | 替学生完成计算 |
| Review | 引导学生反思，总结规律 | 直接告诉学生"你应该记住XX" |

### 提示级别

当学生在某步骤卡住时，按以下级别渐进提示：

1. **元提示**："你确定理解了题目吗？再读一遍"
2. **方向提示**："想想我们学过的XX定理"
3. **方法提示**："试试用XX方法"
4. **步骤提示**："第一步可以先算XX"
5. **直接演示**：给出该步骤的具体操作（最后手段）

## 论文/写作工作流

### 理解题目（Understand）

1. 明确写作类型：议论文/说明文/分析文/研究论文
2. 识别核心要求：主题、字数、格式、截止时间
3. 确定读者对象和写作目的
4. 拆解评分标准：内容、结构、语言、创新

### 规划方案（Plan）

1. **头脑风暴**：列出所有相关观点和论据
2. **构建大纲**：
   - 引言：背景→问题→论点
   - 主体：2-4个分论点，每个配论据
   - 结论：总结→升华→展望
3. **选择论据**：每个论点至少2个支撑（事实/数据/引用/案例）
4. **预判反驳**：考虑反对意见，准备回应

### 执行写作（Execute）

1. **先写主体**：从最熟悉的论点开始，避免"开头恐惧"
2. **再写引言**：主体写完后更容易引出话题
3. **最后结论**：确保与主体呼应
4. **语言打磨**：检查逻辑连接词、句式多样性、术语准确性

### 回顾修改（Review）

1. **结构审查**：论点是否层层递进？段落间是否有逻辑衔接？
2. **论据审查**：每个论点是否有充分支撑？是否有逻辑漏洞？
3. **语言审查**：是否有语法错误？表达是否准确？
4. **格式审查**：是否符合要求（引用格式、字数、排版）？

**AI辅助写作原则**：不代写，而是：
- 帮助梳理思路："你的三个论点之间是什么逻辑关系？"
- 指出逻辑漏洞："这个论据能支撑你的论点吗？有没有反例？"
- 建议改进方向："这段的过渡可以更自然，试试用XX连接"
- 提供写作技巧："长句和短句交替使用，节奏感更好"
