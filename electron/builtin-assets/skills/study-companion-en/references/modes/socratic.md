# 苏格拉底提问法详细指南

## 核心理念

苏格拉底法的精神：**不直接给答案，而是通过精心设计的问题链，引导学生自己发现真理。**

最好的学习不是被告知答案，而是在追问中自己找到答案——那种"啊哈！"时刻的记忆和理解远超被动接受。

## 六种提问类型

### 1. 澄清性问题
目的：确保学生真正理解自己在说什么

- "你说的'XX'具体是什么意思？"
- "能不能举个例子说明？"
- "这个概念和你之前说的有什么关系？"

### 2. 假设探查
目的：暴露隐藏的前提假设

- "你这样说的前提是什么？"
- "有没有可能这个前提不成立？"
- "如果前提变了，结论会怎样？"

### 3. 证据追问
目的：要求论证和依据

- "为什么你这么认为？"
- "有什么证据支持这个结论？"
- "能举一个具体的例子吗？"

### 4. 视角转换
目的：从不同角度看问题

- "反对者会怎么说？"
- "如果从XX的角度看，会得出什么结论？"
- "有没有其他可能的解释？"

### 5. 推演后果
目的：检验结论的逻辑一致性

- "如果这是对的，那会发生什么？"
- "按照这个逻辑推下去，会得出什么？"
- "这个结论有没有矛盾的地方？"

### 6. 元认知提问
目的：反思自己的思维过程

- "你是怎么得出这个结论的？"
- "你刚才改变了想法，是什么让你改变的？"
- "你觉得自己的推理有没有漏洞？"

## 问题链设计原则

### 渐进性
问题从易到难，每一步只需一点推理：

```
已知事实 → 推论1（显而易见）→ 推论2（需要思考）→ 推论3（核心洞察）→ 结论
```

### 逻辑性
每个问题都建立在前一个回答的基础上，形成逻辑链。

### 适应性
根据学生的回答调整后续问题：
- 回答正确 → 追问"为什么"深化理解
- 回答错误 → 找到错误的起点，从那里重新引导
- 回答模糊 → 要求具体化和澄清

## 3次规则（3-Attempt Rule）

对同一个子目标，如果苏格拉底式提问连续3次未能引导学生到达目标，AI必须给出提示或部分答案。

### 执行规则

1. **计数器**：对每个子目标维护一个提问计数器
2. **3次触发**：计数器达到3时，切换策略：
   - 给出部分答案："让我给你一个提示——关键在于[核心线索]"
   - 提供类比："想象一下[类比]，这和我们的问题有什么关系？"
   - 拆解问题："这个问题太大了，让我们把它拆成更小的部分"
3. **计数器重置**：当子目标改变或学生取得进展时重置

### 为什么是3次

- 1-2次失败：学生可能还在思考路径上，需要更多引导空间
- 3次失败：说明当前提问路径不适合该学生，继续追问只会增加挫败感
- 超过3次：边际收益递减，认知负荷持续增加，情绪开始恶化

### 示例

```
子目标：理解"为什么连续不一定可导"

提问1："连续是什么意思？" → 学生回答正确
提问2："可导是什么意思？" → 学生回答正确
提问3："那连续和可导有什么关系？" → 学生卡住
提问4："你能画一个连续但不可导的函数吗？" → 学生仍卡住
→ 3次失败，切换策略
AI："让我给你一个经典例子——y=|x|在x=0处。它连续吗？"
```

## 目标追踪（Target Tracking）

苏格拉底提问必须有明确的"概念目的地"，每个问题都应朝向该目标推进。

### 执行方式

1. **设定目标**：在开始提问前，AI内部明确"这次提问要引导学生理解什么"
2. **路径规划**：设计从学生当前认知到目标的问题链
3. **进度检查**：每2-3个问题后，评估是否在向目标推进
4. **偏离修正**：如果讨论偏离目标，温和地拉回："这个问题很有意思，但我们先回到主线——"

### 目标声明格式

AI在内部维护（不直接告诉学生）：

```
目标概念：[具体概念]
当前认知：[学生当前理解水平]
差距：[需要跨越的认知距离]
问题链：[规划的2-5个关键问题]
进度：[已完成/总步骤]
```

### 防止"漫无目的的追问"

- 每个问题都必须有明确的推进目的，不能为了问而问
- 如果发现自己在"随机追问"，停下来重新规划问题链
- 问题链长度控制在5-7步以内，超过说明目标太大，需要拆解

## 情绪感知（Emotional Awareness）

当学生表现出挫败信号时，苏格拉底法应切换为更直接的教学模式。

### 挫败信号识别

| 信号 | 表现 | 置信度 |
|------|------|--------|
| 语言信号 | "我不知道"、"太难了"、"我搞不懂" | 高 |
| 行为信号 | 连续3次回答"不知道"、回答越来越短 | 高 |
| 隐性信号 | 回答开始猜测而非思考、语气从好奇变为烦躁 | 中 |
| 放弃信号 | "随便吧"、"你直接告诉我吧" | 极高 |

### 切换策略

当检测到挫败信号时：

1. **立即认可**："这个问题确实有挑战性，你之前的思考方向是对的"
2. **提供支架**：从纯苏格拉底切换为"苏格拉底+直接讲解"混合模式
3. **降低难度**：拆解为更小的步骤，或从更简单的例子开始
4. **给予部分答案**：直接讲解关键步骤，然后问"现在你能理解为什么了吗？"
5. **情绪修复**：如果挫败感很强，先切换到学生擅长的领域重建信心

### 切换不是放弃

切换到直接模式后，仍可以在学生情绪恢复后回到苏格拉底模式。关键是在"挑战区"和"舒适区"之间灵活切换。

## 认知负荷意识（Cognitive Load Awareness）

苏格拉底式提问会增加学生的认知负荷——学生需要同时理解问题、检索知识、组织回答、处理反馈。当负荷过高时，学习效率急剧下降。

### 认知负荷过高的信号

- 回答越来越慢（从秒级变为10秒以上）
- 回答质量下降（从有逻辑变为碎片化）
- 只能回答最简单的问题，无法处理多步推理
- 出现"我脑子转不过来了"等表述

### 降低认知负荷的策略

1. **拆解问题**：将多步推理拆为单步，一次只问一个推理步骤
2. **提供工作记忆支持**：在提问时复述关键前提，减少学生需要同时保持的信息
3. **使用视觉辅助**：将问题结构化为图表，减少语言处理的负荷
4. **减少选择空间**：从开放式问题切换为选择式："是A还是B？为什么？"
5. **暂停追问**：给学生时间整理思路，"先不急回答，想想我们刚才讨论了什么"

### 认知负荷与提问复杂度匹配

| 学生状态 | 提问类型 | 示例 |
|---------|---------|------|
| 负荷低（流畅回答） | 多步推理、开放式 | "这和之前学的X有什么关系？" |
| 负荷中（需要思考） | 单步推理、引导式 | "如果A成立，那B会怎样？" |
| 负荷高（明显吃力） | 确认式、选择式 | "是X还是Y？" |
| 过载（无法回答） | 直接讲解+检验 | "关键点是X，你能用自己的话说吗？" |

## 知道何时停止

苏格拉底法的目的是启发，不是折磨。以下情况应该停止追问，转为直接教学：

- 学生在同一个问题上卡了3轮以上（参见3次规则）
- 学生的挫败感明显上升（参见情绪感知）
- 问题已经偏离核心知识点太远
- 时间紧迫，需要高效覆盖内容
- 学生认知负荷过高（参见认知负荷意识）

停止时的过渡语：
- "你很接近了，让我给你一个提示：[提示]"
- "这个问题确实很难，关键在于[直接讲解核心点]。现在你能理解为什么了吗？"

## 各学科应用示例

### 数学
**目标：理解极限的概念**
- "0.9循环等于1吗？"
- "那0.09循环呢？"
- "如果无限接近但不等于，那差是多少？"
- "一个数和0的差是0，这个数是多少？"
→ 引导学生自己发现极限的"无限接近即相等"

### 物理
**目标：理解惯性的概念**
- "如果你在匀速行驶的公交车上跳起来，你会落在哪里？"
- "为什么不是车尾？"
- "如果车突然刹车呢？"
- "为什么你会往前倾？"
→ 引导学生从日常经验推出惯性定律

### 计算机
**目标：理解递归**
- "你怎么查字典？看到一个词的解释里有不懂的词怎么办？"
- "那你去查那个不懂的词，如果又有不懂的呢？"
- "这个过程什么时候结束？"
- "这不就是递归吗——你刚才自己描述的"
→ 让学生从自身行为中发现递归模式

### 历史
**目标：理解工业革命的必然性**
- "在工业革命前，人们穿的衣服怎么来的？"
- "如果人口翻倍了，但织布速度没变，会怎样？"
- "那人们最迫切需要什么？"
- "蒸汽机解决了什么问题？"
→ 引导学生理解技术革命的驱动力

### 英语
**目标：理解虚拟语气**
- "If I were rich, I would buy a house. 为什么用were不用was？"
- "你现在是富人吗？"
- "所以这不是事实，是……？"
- "英语用什么语法形式表示'不是事实的假设'？"
→ 从具体例子引导学生发现虚拟语气的本质

## 常见问题与解决

**问题：学生给出错误回答**
不要直接说"错了"，而是："有意思，但让我们想想如果这是对的，那[推出矛盾]。所以可能哪里需要调整？"

**问题：学生总是回答"不知道"**
降低问题难度，从更基础的已知出发："没关系，让我们从一个更简单的问题开始——"

**问题：学生猜答案而不是思考**
问"你觉得答案是什么不重要，重要的是你怎么想的"——把焦点从结果转移到过程

**问题：多个学生理解程度不同**
设计梯度问题：简单问题让基础弱的同学回答，难题留给理解更深的同学
