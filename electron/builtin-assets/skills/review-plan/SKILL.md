# 复习计划

根据考试时间、科目难度和掌握程度，生成科学合理的复习计划。

## Purpose

帮助用户在有限时间内高效安排复习，平衡多门科目，合理分配精力，确保考前完成所有必要复习。

可生成：
- 每日详细计划（时间段 + 任务）
- 冲刺计划（短期高强度）
- 里程碑检查点
- 多科目时间分配表
- 每日任务清单

支持：
- 按剩余天数自动选择策略
- 多门考试按日期排序
- 根据掌握程度调整时间分配
- 动态调整计划

## When to Use

Use this skill when the user asks to:

- 制定复习计划或学习计划
- 安排考试复习时间
- 规划每天学什么
- 分配多门科目的复习时间
- 做冲刺计划
- 评估剩余时间是否够用
- 调整已有计划
- 安排考前最后几天

示例：

- "帮我做一个期末复习计划。"
- "还有 10 天考试，怎么安排？"
- "三门课的复习时间怎么分配？"
- "每天只有 4 小时，够复习吗？"
- "帮我做个考前 3 天冲刺计划。"

## When Not to Use

Do not use this skill when:

- 用户只想梳理某章知识点（→ 使用知识梳理）
- 用户想做练习题（→ 使用练习与测验）
- 用户想分析错题（→ 使用错题与薄弱点）
- 用户想要模拟考试（→ 使用模拟考试与批改）
- 用户想要考前速记材料（→ 使用考前输出）
- 没有任何考试信息（科目、日期、范围）

如果用户说"帮我梳理一下这章的知识点"，那是知识梳理的场景。
如果用户说"出一套模拟卷"，那是模拟考试的场景。

## Inputs

用户可能提供：

- 考试科目（可多门）
- 考试日期或剩余天数
- 每日可用时间（默认 6-8 小时）
- 掌握程度（好/一般/差）
- 薄弱章节或知识点
- 知识清单或错题本文件
- 优先级偏好（先考先复习 / 先弱后强）

## Document and Reference Handling

当用户提供知识清单、错题本或教材时：

1. 先用 Read 读取
2. 根据材料评估各章节的掌握程度
3. 识别高频考点和薄弱环节
4. 将薄弱章节安排更多时间
5. 不编造考试范围、考试日期或考试规则

## Strategy Selection

根据剩余天数自动选择策略：

| 剩余天数 | 策略 | 特点 |
|----------|------|------|
| ≥14 天 | 系统复习 | 全面覆盖，循序渐进，每章有充足时间 |
| 7-13 天 | 重点突破 | 优先高频考点和薄弱章节，略过低频内容 |
| 3-6 天 | 冲刺模式 | 只攻核心考点 + 错题回顾 + 模拟练习 |
| 1-2 天 | 考前速记 | 速记页 + 公式卡 + 易错清单，不做新题 |

### 策略自动选择

```markdown
你还有 {X} 天，我建议采用 **{策略名称}**：

{策略说明}

如果你有特别薄弱的章节，我会优先安排。
```

## Time Allocation Rules

### 黄金时间原则

- 上午 8:00-11:00：安排最难的科目（大脑最清醒）
- 下午 14:00-17:00：安排中等难度科目
- 晚上 19:00-21:00：安排记忆类任务或轻松科目
- 睡前 30 分钟：回顾当天内容

### 科目权重

| 掌握程度 | 时间权重 | 策略 |
|----------|----------|------|
| 差 | ×2.0 | 优先安排，分配最多时间 |
| 一般 | ×1.5 | 正常安排，重点突破 |
| 好 | ×1.0 | 保持状态，少量复习 |

### 多科目排序

- 先考的科目优先复习
- 同一天考的科目穿插安排
- 难度大的科目安排在状态好的时段
- 每天至少复习 2 门，避免单科疲劳

## Workflow

1. 收集考试信息（科目、日期、范围、掌握程度）
2. 计算剩余天数
3. 选择复习策略
4. 按科目分配时间权重
5. 安排每日具体任务（时间段 + 产出物）
6. 设置里程碑检查点
7. 检查计划的可执行性
8. 输出完整计划

## Quality Criteria

好的复习计划必须：

- 具体到时间段和产出物
- 考虑用户的实际可用时间
- 难度大的科目安排在上午
- 每天留 1 小时做当日回顾
- 考前最后一天不做新题
- 多门考试按日期排序，先考先复习
- 计划现实可执行，不过度乐观
- 包含里程碑检查点
- 考虑休息和睡眠

## Constraints

- 不制定不切实际的计划（如"一天复习完一本书"）
- 不忽略用户的实际时间限制
- 不把所有科目堆在最后一天
- 不安排连续高强度复习超过 3 天（需要缓冲）
- 不忘记安排休息时间
- 考前最后一天不安排新题
- 不编造考试范围或考试规则

## Style Guidelines

默认风格：
- 务实
- 可执行
- 结构化
- 鼓励性
- 灵活

优先使用：
- 清晰的时间表
- 具体的任务描述
- 可检查的产出物
- 合理的难度梯度

避免：
- 过于理想化的时间安排
- 模糊的任务描述（如"复习数学"）
- 忽略休息的高强度计划
- 不考虑用户实际情况的模板化方案

User:

```text
还有 10 天考期末，考高数、线代、概率论，每天能学 6 小时。
```

### 示例 3：冲刺场景

User:

```text
明天就考高数了，帮我安排最后一天。
```

Assistant:

```markdown
好的，考前一天不做新题，专注回顾和巩固。

# 高数考前 1 天安排

## 上午（3 小时）
- 08:00-09:30：翻看速记页和公式卡
- 09:45-11:00：回顾错题本中的高频错题

## 下午（2 小时）
- 14:00-15:30：过一遍易错清单
- 15:30-16:00：默写核心公式

## 晚上
- 19:00-20:00：轻松翻看笔记
- 20:00 后：准备考试用品，22:00 前睡觉

## 注意
- 不做新题
- 不纠结难题
- 保持信心，早睡早起
```

## 追问支持

- "今天没完成怎么办？" → 调整后续计划
- "想加一门课" → 重新平衡时间
- "这章太难了" → 增加该章时间
- "计划太紧了" → 降低目标，聚焦核心
