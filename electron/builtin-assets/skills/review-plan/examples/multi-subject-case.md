# 多科目计划案例

## 用户输入

> 三门课要考：高数6月20号，英语6月22号，政治6月25号，每天能学8小时

## 处理过程

### 1. 信息解析
| 科目 | 考试日期 | 剩余天数 |
|------|----------|----------|
| 高数 | 6月20日 | 15天 |
| 英语 | 6月22日 | 17天 |
| 政治 | 6月25日 | 20天 |

### 2. 策略选择
按考试先后排序：高数 → 英语 → 政治

### 3. 时间权重计算
| 科目 | 难度系数 | 紧急系数 | 权重 | 每日时间 |
|------|----------|----------|------|----------|
| 高数 | 1.5（难） | 1.5（最先考） | 2.25 | 3.5h |
| 英语 | 1.0（一般） | 1.0 | 1.0 | 1.5h |
| 政治 | 0.7（易） | 0.7（最后考） | 0.49 | 1.0h |
| 弹性 | | | | 2.0h |

### 4. 计划框架
- **Phase 1（Day 1-15）**：高数为主（3.5h/天）+ 英语（1.5h/天）+ 政治（1h/天）
- **Phase 2（Day 16-17）**：高数已考完，英语为主（5h/天）+ 政治（2h/天）
- **Phase 3（Day 18-20）**：英语已考完，全力政治（6h/天）
- **最后 1 天**：政治考前速记

### 5. 生成输出
使用 `daily-plan-template.md` 模板，按 Phase 分段生成计划。
