# 信息收集案例

## 场景

用户提供的信息不完整，需要主动询问。

## 用户输入

> 帮我做个复习计划

## 处理过程

### 1. 识别缺失信息
用户未提供：
- ❌ 考试科目（必须）
- ❌ 考试时间（必须）
- ❌ 每天可用时间（可选，有默认值）

### 2. 主动询问

```
好的，我来帮你制定复习计划。请告诉我以下信息：

**必须提供：**
1. 要考哪些科目？
2. 每门课的考试日期是什么时候？

**可选信息（不提供则用默认值）：**
3. 每天能学几个小时？（默认 6-8 小时）
4. 有没有已经整理好的知识清单或错题本？
5. 哪些科目你觉得比较难/没把握？
```

### 3. 用户补充

> 高数6月20号，英语6月22号，每天能学6小时，高数比较难

### 4. 继续执行
有了足够信息，开始制定计划。
