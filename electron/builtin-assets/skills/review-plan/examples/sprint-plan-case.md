# 冲刺计划案例（≤6天）

## 用户输入

> 明天考操作系统，帮我做个冲刺计划

## 处理过程

### 1. 信息解析
- 科目：操作系统
- 剩余天数：1天
- 每日时间：未指定，假设全天可用（约 8 小时）

### 2. 策略选择
剩余 1 天 → 选择"考前速记"策略

### 3. 生成输出
使用 `sprint-plan-template.md` 模板，生成 1 天冲刺计划。

### 4. 输出示例

```
# 操作系统 考前1天冲刺

## ⚡ 核心原则
- 只看速记页、公式卡、易错清单
- 不做新题
- 保证充足睡眠

## 今日安排

| 时间段 | 任务 | 时长 |
|--------|------|------|
| 09:00-10:00 | 速记页过一遍 | 1h |
| 10:00-11:00 | 高频考点清单逐条确认 | 1h |
| 11:00-12:00 | 易错清单扫一遍 | 1h |
| 14:00-15:00 | 错题本只看标记题 | 1h |
| 15:00-16:00 | 公式/概念默记 | 1h |
| 16:00-17:00 | 放松 + 准备考试用品 | 1h |
| 21:00-22:00 | 速记页最后扫一遍 | 1h |
| 22:00 | 睡觉 | |

## 考场策略
- 先做会做的，跳过不会的
- 简答题写关键词拿步骤分
- 最后留 10 分钟检查
```
