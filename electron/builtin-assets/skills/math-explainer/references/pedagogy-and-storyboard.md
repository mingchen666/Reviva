# 讲解设计 & 分镜方法

视频/网页好不好，70% 取决于**讲解设计**，30% 才是技术。先想清楚“怎么讲”，再动手。

---

## 0. 致命红线：讲「为什么」，不要只展示「是什么」（最重要）

这是数学讲解最容易失败、也最致命的地方：**做出来动画很流畅、很好看，但看完根本不知道这个结论是怎么来的。** 这种视频是不合格的，必须重做。

### 反例 vs 正例（看清差别）

> **反例（结论展示陷阱）**：画一个直角三角形 → 长出两个正方形写着 a²、b² → 蹦出公式 `a²+b²=c²` → 收尾。
> 观众只看到“有这么个结论”，**完全不知道为什么成立**。漂亮 ≠ 讲清楚。

> **正例（演示推导）**：两个一样大的正方形 → 各塞进 4 个相同的直角三角形 → 左边空出 a²+b²、右边空出 c² → “大正方形相等、扣掉的三角形相等，所以剩下的相等” → `a²+b²=c²`。
> 观众**亲眼看着结论被推出来**。画面本身就是论证。

完整正例见 `templates/example_pythagoras_proof.py`（重排/面积法证明，已验证可渲染）。**做证明类视频前，先读它。**

### 验收测试（动手前后都要自问）

- **遮住旁白测试**：把字幕/旁白全删掉，只看画面，观众还能复述出推理的关键步骤吗？能 → 合格；不能 → 画面没有承载论证，重新设计。
- **“凭什么”测试**：推导的每一步，观众都能回答“为什么这一步成立”吗？任何一步是“凭空出现/直接给结论”，就是断层，必须补上中间步骤。
- **时间预算**：**推导/论证过程至少占全片 50%**。钩子+收尾是配角，推导才是主角。不要把时间浪费在炫酷开场而推导一笔带过。

### 怎么设计“推导”这一核心段落

1. **先选定一种推导/证明路径**（见下方 §7 策略库），不要含糊。
2. **拆成原子步骤**：每一步只前进一个最小的逻辑单位。
3. **让画面切换 = 一步逻辑推理**：每次动画都对应“因为……所以……”，而不是单纯的装饰运动。
4. **每步配一句“为什么”的旁白**，把视觉变化翻译成逻辑。
5. **关键等式/不等式要可见地被“做”出来**（面积拼合、变形对齐、逼近过程），而不是直接写出答案。

## 1. 把概念讲清楚的 5 条原则

1. **一个视频只讲一个核心思想。** 贪多必乱。砍到只剩一句话能说清的目标。
2. **从具体到抽象。** 先给一个能看见、能摸到的例子（一个具体三角形、一个具体函数），再抽象成符号。
3. **制造“疑问→解答”的张力。** 先抛出问题或反直觉现象，让观众想知道答案，再揭示。
4. **可视化优先于文字。** 能用图形/动画表达的，就不要堆文字。文字只做点睛。
5. **每步都回答“所以呢”。** 每个画面都要服务于核心思想，否则删掉。

## 2. 经典讲解结构（套用即可）

```
① 钩子（Hook，3–8s）：一个问题 / 一个惊奇现象 / 一个具体场景
② 直观（Intuition，20–40s）：用具体例子和动画建立直觉
③ 形式化（Formalize，20–40s）：把直觉写成公式 / 定理 / 步骤
④ 验证或推导（Why，20–40s）：为什么成立（一个证明片段 / 一次演算）
⑤ 收尾（Recap，5–10s）：一句话总结 + 一个延伸思考 / 应用
```

- 视频：按这 5 段排镜头。
- 交互网页：①②放顶部讲解区，③公式区，④⑤靠用户拖动控件自己“发现”。

## 3. 分镜（storyboard）怎么写

分镜是给代码的施工图。**每个镜头**回答三件事：

1. **画面**：屏幕上有什么？（对象、位置、颜色）
2. **旁白/字幕**：这一镜要说的一句话。
3. **动作**：用什么动画进入、如何切到下一镜（FadeOut 谁、Transform 成谁）。

写法见 `templates/storyboard.md`，复制到工作区填。

**镜头粒度建议**：一个镜头 = 一个完整小意思，时长 3–10s。整片 6–12 个镜头较合适。

---

## 4. 视觉与节奏规范

- **配色**：深色背景（默认）+ 1 个主色（强调）+ 1 个辅色（对比）+ 白/灰（文字）。别超过 3 种强色。
- **一次只动一处**：观众视线有限，避免同屏多个无关动画抢注意力。
- **停留时间**：字幕出现后至少留“能读完 ×1.5”的时间，用 `self.wait()`。公式更久。
- **强调手法**：要点出现用 `Indicate / Circumscribe / Flash`，颜色变亮，或缩放一下。
- **进入/退出一致**：同类元素用同一种进出动画，建立视觉语言。

---

## 5. 常见反面教材（避免）

- 一屏塞满公式和文字，观众无从看起。→ 拆镜头、做减法。
- 旁白和画面不同步（说 A 画面在 B）。→ 分镜里把旁白绑定到镜头。
- 只有结论没有“为什么”。→ 至少给一段直观推导。
- 动画炫技但不服务理解（无意义旋转/粒子）。→ 删。
- 公式直接砸出来不解释每个符号。→ 逐项引入或高亮。

## 6. 一个填好的分镜示例（勾股定理 · 演示推导版）

注意：**核心是镜头 3–6 的推导**，而不是直接展示公式。对应代码见 `templates/example_pythagoras_proof.py`。

| # | 阶段 | 画面 | 旁白/字幕 | 动作 |
|---|------|------|-----------|------|
| 1 | 问题 | 直角三角形，标 a、b、c | “两条直角边 a、b，斜边 c，为什么 a²+b²=c²？” | Create + 标注 |
| 2 | 铺垫 | 两个等大的正方形（边长 a+b） | “画两个一样大的正方形，面积相等。” | Create×2 |
| 3 | **推导** | 左方形塞入 4 个三角形，空出 a²、b² | “塞进 4 个相同三角形，左边剩下 a² 和 b²。” | 三角形飞入 + 高亮空白 |
| 4 | **推导** | 右方形塞入同 4 个三角形，空出 c² | “同样 4 个三角形换个放法，右边只剩 c²。” | 三角形飞入 + 高亮 |
| 5 | **推导(关键)** | 两图并排，高亮两处空白 | “大正方形相等、扣掉的三角形相等 → 剩下必相等。” | Indicate 两处空白 |
| 6 | 结论 | 三步逻辑 + `a²+b²=c²` 加框 | “所以 a²+b²=c²。” | 逐条列出 + 定格 |

把它翻译成 `scene.py`：直接参考 `example_pythagoras_proof.py` 的 `intro / build_two_squares / fill_left / fill_right / compare_and_conclude` 结构。

## 7. 常见定理/概念的「推导策略」库

不知道怎么演示“为什么”时，从这里选一条已被验证好讲的路径。每条都强调**让画面把结论“做”出来**。

| 主题 | 推导策略（怎样让画面承载论证） |
|------|--------------------------------|
| 勾股定理 | **重排/面积法**：两等大正方形塞同样 4 个三角形，空白从 a²+b² 变 c²（见范例文件） |
| 三角形内角和=180° | 过顶点作底边平行线，用**内错角**把三个角平移拼成一条直线 |
| 圆面积 πr² | 把圆切成许多扇形，**交错拼成近似矩形**（长≈πr，宽≈r），扇形越多越像矩形 |
| 等差数列求和 | **倒序相加**：正着写一行、倒着写一行，逐项配对都等于(首+尾)，可视化成等高条 |
| 平方差 a²−b² | 大正方形挖去小正方形，把 L 形**剪开重拼成 (a+b)×(a−b) 矩形** |
| 完全平方 (a+b)² | 边长 a+b 的正方形**切成 4 块**：a²、b²、两个 ab，面积相加 |
| 导数=切线斜率 | 割线两点，**让一个点沿曲线趋近另一个点**（ValueTracker），割线变切线 |
| 定积分=面积 | **黎曼和**：矩形条逐渐变窄变多，阶梯逼近曲线下面积 |
| 相似三角形 | 一个三角形**等比缩放**叠到另一个上，对应边比值恒定可视化 |
| 正弦函数来源 | 单位圆上动点，**纵坐标随角度的变化**实时描成 sin 曲线（动点联动） |
| 三角形面积=½底×高 | 复制一份旋转**拼成平行四边形/矩形**，面积减半 |
| 韦达定理 | 由 (x−x₁)(x−x₂) 展开，**逐项对齐**系数得到根与系数关系 |

选定策略后，按 §0 的“怎么设计推导”拆成原子步骤，再落到分镜表。
