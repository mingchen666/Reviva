# 考前输出

在考前最后阶段，输出高密度、可快速浏览的冲刺材料。

## Purpose

考前时间紧迫，需要最高信息密度的复习材料。这个技能专注于生成可以快速翻阅、反复过目的冲刺资料，帮助用户在最后阶段最大化复习效果。

可输出：
- 速记页 — 核心知识点压缩版
- 公式卡 — 公式 + 适用条件 + 易错标注
- 易错清单 — 正误对比 + 避免方法
- 高频考点 — 历年高频知识点排序
- 答题模板 — 各题型的标准答题框架
- 考前冲刺安排 — 时间规划 + 优先级

## When to Use

Use this skill when the user asks to:

- 生成考前速记材料
- 整理公式卡或定理卡
- 列出易错点清单
- 总结高频考点
- 做答题模板
- 安排考前冲刺
- 帮我速记或考前突击
- 明天考试帮我准备
- 考前 2 小时 / 1 天的安排

示例：

- "明天考高数，帮我做个速记页。"
- "整理一下概率论的公式卡。"
- "帮我列一下这门课的易错清单。"
- "考前 2 小时看什么最有用？"
- "帮我做一个英语作文的答题模板。"

## When Not to Use

Do not use this skill when:

- 用户想从头学习新知识（→ 使用知识梳理）
- 用户想做练习题（→ 使用练习与测验）
- 用户想分析错题（→ 使用错题与薄弱点）
- 用户想要完整的模拟考试（→ 使用模拟考试与批改）
- 用户想制定长期复习计划（→ 使用复习计划）
- 距离考试还有很长时间（>7 天，建议使用复习计划）

考前输出专注于"最后阶段的回顾和强化"，不是学习新内容的工具。

## Inputs

用户可能提供：

- 考试科目
- 材料类型（速记页/公式卡/易错清单/高频考点/答题模板）
- 剩余时间（2 小时 / 1 天 / 自定义）
- 知识清单或错题本文件路径
- 教材或讲义文件
- 考试范围或重点章节

## Missing Information Handling

### 未指定材料类型

```markdown
可以帮你准备考前材料。请告诉我：

1. 考什么科目？
2. 还剩多少时间？（如"明天考试"、"还有 2 小时"）
3. 有没有知识清单或错题本？

如果没指定想要什么材料，我会根据剩余时间自动安排：
- 考前 2 小时：速记页 + 公式卡
- 考前 1 天：速记页 + 公式卡 + 易错清单 + 冲刺安排
```

### 只说了"明天考试"

```markdown
好的，明天考试。请告诉我：

1. 考什么科目？
2. 有没有知识清单、错题本或讲义文件？
3. 你最担心哪部分？

我会帮你准备考前 1 天的冲刺材料。
```

### 没有提供任何材料

```markdown
没有材料文件的话，我只能生成框架性的速记内容。

为了更准确，建议你提供：
- 知识清单或笔记文件
- 错题本
- 教材或讲义

如果都没有，告诉我科目和章节，我会基于通用知识生成，但可能不够精准。
```

## Document and Reference Handling

当用户提供材料时：

1. 先用 Read 读取所有相关文件
2. 从材料中提取最核心的内容
3. 压缩到最高信息密度
4. 保留公式、定义、定理、易错点
5. 删除解释性文字和背景知识
6. 以快速浏览为首要目标
7. 不编造材料中没有的知识点

## Material Type Selection

根据剩余时间自动选择：

| 剩余时间 | 必出 | 推荐 | 可选 |
|----------|------|------|------|
| 2 小时 | 速记页 | 公式卡 | — |
| 1 天 | 速记页 | 公式卡 + 易错清单 | 冲刺安排 |
| 2-3 天 | 速记页 | 公式卡 + 易错清单 | 高频考点 + 答题模板 |
| 3-7 天 | — | — | 全部可用 |

## Compression Rules

速记页的压缩原则：

- 100% 内容 → 压缩到 5-10% 体积
- 保留：公式、定义、定理、关键结论、易错点
- 删除：推导过程、背景介绍、详细解释、例题
- 使用缩写和符号
- 表格 > 列表 > 段落
- 每个知识点不超过 2 行

### 公式卡规则

每个公式包含：
- 公式本身
- 适用条件（什么时候能用）
- 变形（常见变化形式）
- 易错标注（常见错误用法）
- 常考题型

### 易错清单规则

每个易错点包含：
- 错误描述（常见错误是什么）
- 正确做法（应该怎么做）
- 正误对比（一眼看出区别）
- 记忆口诀（如有）

## Workflow

1. 读取已有资料（知识清单、错题本、讲义）
2. 识别考试科目和剩余时间
3. 确定输出材料类型
4. 从资料中提取核心内容
5. 压缩到目标信息密度
6. 按模板格式化输出
7. 检查完整性

## Auto-Combination

用户说"帮我准备明天的考试"时，自动输出：
1. 速记页（必出）
2. 公式卡（理科必出）
3. 易错清单（如有错题本）
4. 考前 1 天冲刺安排

## Quality Criteria

好的考前材料必须：

- 信息密度极高，无冗余
- 使用表格、列表等便于快速浏览的格式
- 公式卡标注适用条件和易错点
- 易错清单有正误对比
- 有错题本/知识清单时优先从中提取
- 无资料时生成框架，提示用户补充
- 适合反复翻阅
- 考前 2 小时不引入新概念

## Constraints

- 不在考前材料中加入大量解释性文字
- 不引入用户材料中没有的新概念
- 不生成练习题（考前不做新题）
- 不生成过于详细的内容（那是知识梳理的范畴）
- 不编造公式、定义或知识点
- 不在考前 2 小时的材料中放太多内容（聚焦核心）

## Style Guidelines

默认风格：
- 高密度
- 简洁
- 直观
- 易扫
- 实用

优先使用：
- 表格对比
- 符号和缩写
- 标红和标注
- 清晰的层级
- 正误对比

避免：
- 长段落
- 详细解释
- 推导过程
- 背景知识
- 冗余信息
