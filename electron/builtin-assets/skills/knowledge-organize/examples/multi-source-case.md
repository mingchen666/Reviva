# 多份资料整合的处理案例

## 用户输入

> 我有三个文件要一起整理：`操作系统教材.pdf`、`课堂笔记.md`、`老师划的重点.txt`

## 处理过程

### 1. 逐一读取
按顺序读取三个文件：
- `操作系统教材.pdf` — 内容最全面，作为主体框架
- `课堂笔记.md` — 补充教材中没有的要点
- `老师划的重点.txt` — 用于调整重要程度标注

### 2. 整合策略
1. 以教材的章节结构为骨架
2. 用课堂笔记补充遗漏的知识点
3. 用老师划的重点调整星级标注：
   - 老师明确划的重点 → 至少 ⭐⭐
   - 老师说"必考" → ⭐⭐⭐
   - 老师说"了解即可" → ⭐

### 3. 冲突处理
当不同资料对同一知识点的描述有差异时：
- 以教材为准（权威性最高）
- 在输出中标注：`📝 注：课堂笔记中提到...`

### 4. 输出
使用 **outline-template.md** 生成大纲，在头部标注资料来源：
```markdown
> 资料来源：操作系统教材.pdf、课堂笔记.md、老师划的重点.txt
> 重点标注参考：老师划的重点.txt
```

## 关键原则

- **教材为骨架**：章节结构以教材为准
- **笔记为补充**：课堂笔记补充教材遗漏
- **重点为标尺**：老师划的重点决定星级
- **冲突要标注**：不同来源有差异时如实标注
