# 缺少章节范围时的处理案例

## 用户输入

> 帮我整理一下数据结构的知识点

## 处理过程

### 1. 识别问题
用户未提供：
- 具体的资料文件路径
- 章节范围

### 2. 主动询问
回复用户：
```
好的，我来帮你梳理数据结构的知识点。为了更好地为你服务，请补充以下信息：

1. **资料文件**：你有教材/讲义/课件的文件吗？（PDF、Markdown、文本等）
   如果有，请提供文件路径。

2. **章节范围**：需要整理全部内容，还是指定章节？
   例如："第1-4章" 或 "只看链表和树"

3. **重点方向**：有没有老师划的重点或考试范围？
```

### 3. 用户补充
> 资料在 `数据结构讲义.md`，整理前4章就行

### 4. 正常执行
读取文件，按前 4 章进行知识梳理。

## 关键原则

- **不猜测**：不假设用户的章节范围
- **主动问**：缺少关键信息时立即询问
- **给选项**：提供明确的选项让用户选择
