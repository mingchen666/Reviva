# 练习与测验

按章节、题型、难度灵活生成练习题，附带答案与详细解析。

## Purpose

帮助用户通过做题巩固知识、检测掌握程度、发现薄弱环节。不仅仅是出题，更重要的是通过练习促进学习。

可生成：
- 选择题（单选 / 多选）
- 填空题
- 判断题
- 简答题
- 计算题
- 分析题
- 编程题
- 综合混合题型

支持：
- 即时答案 + 解析
- 先做后批改模式
- 难度分级
- 按知识点出题
- 干扰项设计

## When to Use

Use this skill when the user asks to:

- 生成练习题或测验
- 按章节出题
- 按题型出题
- 刷题或做题巩固
- 检测对某知识点的掌握
- 做课后练习
- 生成答案和解析
- 生成干扰项或陷阱题
- 按难度出题（基础/中等/拔高）

示例：

- "给我出 10 道高数选择题。"
- "按第三章内容出一套练习。"
- "出几道微积分的计算题，中等难度。"
- "帮我做这章的课后练习。"
- "出 5 道填空题测测我对线性代数的掌握。"

## When Not to Use

Do not use this skill when:

- 用户只想理解某个概念，不想做题（→ 使用知识梳理或概念解释）
- 用户想分析已有的错题（→ 使用错题与薄弱点）
- 用户想要完整的模拟考试卷（→ 使用模拟考试与批改）
- 用户想制定复习计划（→ 使用复习计划）
- 用户想要考前速记材料（→ 使用考前输出）
- 没有任何科目、章节或材料作为出题范围

如果用户说"出一套期末模拟卷"，那是模拟考试的场景，应使用模拟考试与批改。
如果用户说"这几道题我做错了，帮我分析"，那是错题分析的场景，应使用错题与薄弱点。

## Inputs

用户可能提供：

- 科目 / 章节 / 知识点范围
- 教材、讲义、笔记等材料文件
- 题型要求（选择/填空/判断/简答/计算/分析/编程）
- 数量要求（默认 5-10 道）
- 难度要求（基础/中等/拔高/综合，默认中等）
- 是否需要答案和解析
- 是否先做后批改
- 特定知识点或考点的针对性出题

## Document and Reference Handling

当用户提供材料时：

1. 先用 Read 读取所有相关文件
2. 以用户提供的材料为出题的主要依据
3. 优先考查材料中的核心知识点
4. 使用材料中的术语、公式、定义
5. 不编造材料中没有的知识点或事实
6. 材料不完整时，基于已有内容出题并标注覆盖范围有限
7. 如果用户指定了章节，严格在该章节范围内出题

## Question Design Principles

### 出题原则

- 覆盖核心知识点，不重复考同一点
- 题目表述清晰无歧义
- 难度与用户要求匹配
- 测试理解而非单纯记忆（除非用户要求）
- 干扰项有合理的设计逻辑
- 不出偏题、怪题（除非用户要求）

### 干扰项设计（选择题）

好的干扰项应该：
- 基于常见错误理解设计
- 与正确答案形式相近
- 具有一定迷惑性
- 每个干扰项都有明确的错误原因

避免：
- 明显不相关的选项
- 所有选项长度差异过大
- 使用"以上都对/以上都不对"作为常规手段

### Bloom 认知层级

| 层级 | 说明 | 对应难度 |
|------|------|----------|
| 记忆 | 识别、回忆基本事实 | 基础 |
| 理解 | 解释、归纳、比较 | 基础-中等 |
| 应用 | 在新情境中使用知识 | 中等 |
| 分析 | 分解、识别关系和模式 | 中等-拔高 |
| 评价 | 判断、论证、批评 | 拔高 |
| 创造 | 设计、构建、提出方案 | 拔高 |

### 难度校准

- 🟢 基础：直接套用公式/定义，一步计算，明确知识点
- 🟡 中等：需要 2-3 步推理，或需要选择合适方法
- 🔴 拔高：综合多个知识点，需要创造性思维，或有陷阱

## Workflow

1. 识别出题范围（科目、章节、知识点）
2. 确定题型、数量、难度
3. 如果提供了材料，先读取并提取核心知识点
4. 设计题目，确保覆盖核心知识点
5. 设计干扰项（选择题）或评分要点（主观题）
6. 编写答案和解析
7. 校验：答案正确、解析清晰、难度匹配、无歧义
8. 分开输出题目和答案

## Quality Criteria

好的练习题必须：

- 事实准确，答案正确
- 符合要求的难度级别
- 与出题范围和材料一致
- 题目表述清晰无歧义
- 选择题干扰项设计合理
- 解析讲清"为什么对"和"为什么错"
- 覆盖核心知识点，不偏不怪
- 有学习价值，促进理解

## Constraints

- 不编造事实、公式、定义或参考文献
- 不超出用户指定的范围出题
- 不把不确定的答案表述为确定的
- 不出过于简单或过于刁钻的题（除非用户要求）
- 不重复考查同一个知识点（除非用户要求）
- 不在题目中使用歧义表述
- 需要联网查最新事实且工具可用时使用搜索
- 搜索不可用时明确说明而非猜测

## Style Guidelines

默认风格：
- 清晰
- 结构化
- 教育性
- 准确
- 实用
- 友好

优先使用：
- 简洁的题目表述
- 清晰的标签和编号
- 便于核对的答案格式
- 强化学习的解析

避免：
- 模棱两可的表述
- 过于复杂的题干
- 没有解析的答案
- 超纲的内容
