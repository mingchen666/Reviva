# 出题原则与质量标准

## 一、通用原则

### 覆盖性
- 同一组题覆盖该章节的核心知识点
- 同一知识点不在同一组题中重复出现（除非刻意对比）
- 优先覆盖高频考点

### 准确性
- 题目表述无歧义
- 答案唯一且正确
- 解析逻辑清晰、有据可依

### 实用性
- 题目风格贴近真实考试
- 难度分布合理（基础60% + 中等30% + 拔高10%）

## 二、各题型设计规范

### 选择题
- 4 个选项（A/B/C/D），1 个正确答案
- 干扰项设计原则：
  - 基于常见错误理解设计
  - 基于易混淆概念设计
  - 基于计算常见失误设计
  - 避免"以上都对/以上都错"等选项
- 题干清晰，不使用否定句（除非考查"不正确的是"）

### 填空题
- 答案唯一、明确
- 空格处填写关键词/数值/公式
- 避免填空位置有多种合理答案

### 判断题
- 表述清晰，避免绝对化用语（"一定"、"所有"、"绝不"）
- 错误项要有明确的错误点
- 正确项不能过于明显

### 简答题
- 指向明确，有标准答案要点
- 评分标准可量化（按要点给分）
- 答案包含：定义 + 特点 + 举例（视情况）

### 计算题
- 给出完整的已知条件
- 所需公式在考纲范围内
- 计算量适中，不超过 3 步核心推导
- 答案可验证

### 编程题（理工科适用）
- 题目描述清晰，输入输出明确
- 边界条件已说明或可推断
- 难度与课内教学匹配

## 三、解析规范

### 选择题解析
```
答案：{正确选项}
解析：本题考查 {知识点}。{解释为什么选这个}。
干扰项分析：
- A：{为什么错}
- C：{为什么错}
- D：{为什么错}
易错点：{提醒}
```

### 计算题解析
```
解题步骤：
1. {已知条件整理}
2. {选用公式及理由}
3. {代入计算过程}
4. {最终结果}

易错步骤：{第几步容易出错，为什么}
```
