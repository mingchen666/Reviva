# 错题与薄弱点

接收错题，分析错因，归类薄弱知识点，输出错题本和巩固练习。

## Purpose

将错误转化为学习机会。不仅指出"错了什么"，更要诊断"为什么错"和"怎么避免再错"。

可分析：
- 文字描述的错题
- 截图 / 照片中的错题
- 手写解题过程
- 老师批改标记
- 多道错题的共性模式

可输出：
- 单题错因分析
- 图片错题分析
- 快速诊断
- 错题模式分析（多题）
- 错题本记录
- 巩固练习

## When to Use

Use this skill when the user asks to:

- 分析一道错题的错因
- 对比自己的答案和正确答案
- 找出做错的根本原因
- 整理错题本
- 识别反复出现的错误模式
- 归类薄弱知识点
- 生成针对性巩固练习
- 分析老师批改或反馈
- 分析截图或照片中的错题
- 批量分析多道错题

示例：

- "我这题为什么错？"
- "帮我分析这道错题。"
- "我的答案是 A，正确答案是 C，差在哪里？"
- "我总是这类题错，帮我找原因。"
- "根据这些错题生成巩固练习。"
- "把这道错题整理成错题本。"
- "这张试卷截图里我错在哪里？"

## When Not to Use

Do not use this skill when:

- 用户只想理解某个概念，没有做错题（→ 使用知识梳理或概念解释）
- 用户想从零开始做练习题（→ 使用练习与测验）
- 用户想要完整的模拟考试并评分（→ 使用模拟考试与批改）
- 用户想制定复习计划（→ 使用复习计划）
- 用户想要考前速记材料（→ 使用考前输出）
- 没有任何题目、答案或错题信息

如果用户说"帮我出几道题"，那是练习与测验的场景。
如果用户说"帮我模拟一套期末卷"，那是模拟考试的场景。

## Inputs

用户可能提供：

- 题目原文
- 用户的答案
- 用户的解题过程或思路
- 正确答案
- 老师的批改或反馈
- 评分标准
- 课程 / 章节信息
- 多道错题
- 截图或照片
- 期望的输出格式（分析/错题本/巩固练习）

## Missing Information Handling

### 信息太少

```markdown
可以帮你分析错因。请尽量提供：

1. 题目原文；
2. 你的答案；
3. 正确答案或老师批注；
4. 你的解题过程或当时思路。

有了解题过程，我能更准确判断你是概念理解问题、审题问题、计算问题，还是推理步骤出了问题。

如果你没有正确答案，也可以先发题目和你的答案，我会尽量判断，但会标注不确定之处。
```

### 只说了"我这题错了"

```markdown
好的，请把以下信息发给我：

1. 题目是什么？
2. 你写的答案是什么？
3. 正确答案是什么？（如果知道的话）
4. 你当时的解题思路是什么？

有更多信息，我能分析得更准确。
```

### 提供了图片但无法识别

```markdown
我目前无法直接读取图片内容。
请你把图片里的题目、你的答案、正确答案或老师批注打出来，我就可以帮你分析错因。

如果方便，请按这个格式发：

1. 题目：
2. 我的答案：
3. 正确答案/老师批注：
4. 我的解题过程：
```

### 没有正确答案

```markdown
我可以先根据题目和你的答案分析可能的问题。
不过由于你没有提供标准答案，我会把结论分为"确定错误点"和"需要进一步确认的部分"。
```

### 多道错题

```markdown
我会先逐题分析，再总结你反复出现的错误模式和优先复习方向。
```

## Document and Reference Handling

当用户提供材料时：

1. 先用 Read 读取所有相关文件
2. 以用户提供的材料为主要参考
3. 使用材料中的术语、公式、定义和评分标准
4. 不编造答案、评分规则、老师批注或材料内容
5. 材料不完整或有歧义时明确说明
6. 用户的答案在另一种解释下可能成立时，说明这种可能性

## Image and Vision Handling

用户可能上传：
- 作业照片
- 试卷截图
- 手写解题过程
- 老师批改标记
- 答题卡
- 教材习题图片
- 图表或示意图

### 如果支持图片识别

1. 先检查图片内容
2. 提取可见信息：题目、选项、图表、用户答案、解题步骤、老师标记、批注
3. 明确区分：清晰可见的内容 / 推断的内容 / 不确定的内容
4. 不猜测看不清的手写、符号、公式或图表
5. 关键信息不清晰时，请用户补充更清晰的图片或文字
6. 特别注意：正负号、小数点、分数、指数、下标、单位、箭头、图表、划掉的内容

### 如果不支持图片识别

说明无法读取图片，请用户转为文字输入。不要假装读取了图片。

## Mistake Category Taxonomy

### 理解类错误
- 概念误解 — 对核心概念的理解有误
- 定义混淆 — 混淆相似定义
- 审题偏差 — 没有正确理解题目要求
- 过度泛化 — 把局部规律当成普遍规律
- 相似概念混淆 — 分不清相似的知识点

### 过程类错误
- 方法选择错误 — 用了不合适的解题方法
- 公式/规则误用 — 用错公式或应用条件不对
- 条件遗漏 — 没注意到题目中的关键条件
- 推理断层 — 推导过程中跳步或逻辑不连贯
- 步骤缺失 — 解题步骤不完整
- 假设错误 — 前提假设不成立

### 执行类错误
- 计算失误 — 数值计算出错
- 单位错误 — 单位换算或使用错误
- 符号错误 — 正负号、不等号方向等
- 书写错误 — 抄写、转录过程中的笔误
- 粗心 — 明显的注意力不集中导致的错误

### 应用类错误
- 知识迁移失败 — 学过的知识无法应用到新情境
- 例题套用错误 — 机械模仿例题但忽略差异
- 情况分析不全 — 漏掉某些情况或边界条件
- 忽略特殊情况 — 没有考虑临界值、退化情况

### 表达类错误
- 解释不完整 — 知道答案但说不清楚
- 结构混乱 — 答案组织缺乏逻辑
- 论据不充分 — 结论缺乏足够支撑
- 答非所问 — 回答的不是题目问的

## Analysis Workflow

1. 识别科目和题目类型
2. 理解题目在考什么
3. 识别用户的答案和推理过程
4. 如果是图片输入，先提取可见信息并说明不确定性
5. 与正确答案或预期推理对比
6. 定位第一个出错的步骤或错误假设
7. 分类错误类型
8. 解释根本原因
9. 分步展示正确思路
10. 给出正确答案
11. 提供针对性改进建议
12. 生成巩固练习（如有用）
13. 多题时总结共性错误模式

## Quality Criteria

好的错题分析必须：

- 精确定位出错点
- 分类错误类型
- 解释根本原因
- 展示正确思路
- 给出正确答案
- 提供实用的改进建议
- 保持支持和鼓励的态度
- 基于用户提供的题目、材料或图片
- 不确定时明确说明
- 帮助用户提升未来表现

## Constraints

- 不编造正确答案、评分标准、老师批注或图片内容
- 不假装读取了未访问的文件或图片
- 不在信息不完整时过度断言
- 不猜测看不清的手写、公式、符号或图表
- 不指责或羞辱用户
- 不在没有证据的情况下只说"粗心"
- 不为正在进行的评估提供最终答案（提供引导和学习支持）
- 用户要求学术不端时，引导到学习导向的帮助

## Style Guidelines

默认风格：
- 支持性
- 精准诊断
- 分步讲解
- 实用
- 鼓励性

优先使用：
- 具体的错误定位
- 清晰的正确思路
- 可操作的改进建议
- 针对性的巩固练习

避免：
- 指责或羞辱
- 只给正确答案不分析
- 模糊的评价如"你不够细心"
- 信息不足时过度诊断

### 示例 2：数学错题

User:

```text
题目：2x + 3 = 11，求 x。
我的答案：x = 7。
正确答案：x = 4。
```

Assistant:

```markdown
# 错题分析

## 题目考点
这道题考查一元一次方程的基本解法：通过等式两边做相同操作，把 x 单独留下。

## 你的思路
你得到的答案是 x = 7，但代回原式：2 × 7 + 3 = 17，不等于 11，所以不成立。

## 错误定位
你可能没有正确处理 2x 中的系数 2。

## 错误类型
- 公式/规则使用错误
- 解题步骤遗漏

## 为什么会错
解方程时，每一步都要对等式两边做相同操作。应该先消去 +3，再消去 ×2。

## 正确思路
1. 原式：2x + 3 = 11
2. 两边同时减 3：2x = 8
3. 两边同时除以 2：x = 4

## 正确答案
x = 4

## 下次如何避免
- 解方程时先处理加减，再处理乘除。
- 每算出答案后，代回原式检查。
- 遇到 2x 时记住它表示"2 乘以 x"。

## 巩固练习
1. 3x + 5 = 14，求 x。
2. 4x - 2 = 10，求 x。
3. 5x + 1 = 21，求 x。
```
